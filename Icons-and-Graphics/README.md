# Icons and Graphics

Icon libraries, illustrations, decorative graphics, and UI/marketing artwork.

## What belongs here

- **Icons** — branded icon sets (functional, not app-launcher icons)
- **Illustrations** — custom brand illustrations, spot art, editorial graphics
- **Patterns & textures** — backgrounds, overlays, repeat patterns
- **Infographics sources** — base templates for data visualization

## Suggested subfolder structure

Add these as you need them:

- `Icons/` — SVG icon set
- `Illustrations/` — spot art, hero illustrations
- `Patterns/` — repeating backgrounds
- `Infographic-Templates/` — chart and diagram starters

## Format guidance

- **SVG** — always the default for icons and simple illustrations
- **PNG** — only when transparency or raster-specific effects needed
- **AI / Figma source** — keep source files in a `Working-Files/` subfolder

## What NOT to put here

- Logos (→ `Logos/`)
- App/favicon assets (→ `Logos/Favicon-App-Icons/`)
- Stock iconography you didn't customize (link externally instead)

## Naming

```
brand-[icon|illustration|pattern]-[name]-v[n].ext
```
