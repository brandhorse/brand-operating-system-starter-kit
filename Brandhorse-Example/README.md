# Brandhorse-Example

A populated reference fork of the Brand Operating System scaffold. This folder shows what each governance layer looks like when filled in — not Brandhorse's full operational state, but representative examples so you can see the *shape* of populated content before you populate your own.

## How to use this folder

**As a reference, not a template to copy.** Every brand's evidence, positioning, voice, and rules will be different. The point of this folder isn't to clone Brandhorse — it's to show you the pattern. Open `00_charter/EXAMPLE.md` and walk down the chain. Notice:

- How upstream decisions explicitly cite the evidence that backs them.
- How the voice is defined alongside the *anti-voice* it stands in contrast to.
- How rules are tagged with severity and the upstream source they enforce.
- How the asset library at the bottom of the chain references the rules that govern it.

## What this is *not*

- Not Brandhorse's complete brand operating system — that's hundreds of evidence rows, dozens of patterns, and a live rule library that compliance runs against. This is a representative slice.
- Not legal/compliance advice for your own brand. Voice, rules, and positioning are specific. Yours will be different.
- Not a static deliverable. The real BOS is a living artifact — evidence accumulates, patterns evolve, rules get refined. These EXAMPLE.md files are frozen at one point in time for teaching.
