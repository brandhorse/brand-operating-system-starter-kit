# Photography

Brand photography, organized by use case.

## Subfolder guide

- **`Lifestyle/`** — on-brand lifestyle imagery (customers, contexts, environments).
- **`Product/`** — product photography (studio, in-context, exploded views, detail shots).
- **`Team/`** — team/founder headshots, office, behind-the-scenes.
- **`Usage-Guidelines/`** — when to use what, licensing, attribution requirements.

## Size guide

Keep at least:

- **Hero-resolution** — 3000px wide or larger for website heroes and print
- **Mid-resolution** — 1200–1600px wide for blog posts, social, slides
- **Thumbnail** — 400–600px wide for index views

For large originals, use Git LFS.

## Licensing

Any stock photography must have license documentation in `Usage-Guidelines/`. Attribute where required.

## Naming

```
brand-photo-[category]-[subject]-v[n]-YYYY-MM.ext
```

Examples:
- `brandhorse-photo-team-jeff-headshot-v1-2026-04.jpg`
- `brandhorse-photo-lifestyle-workspace-hero-v2-2026-03.jpg`
