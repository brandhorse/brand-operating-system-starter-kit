# Changelog

All notable changes to this template are documented here. Semantic-ish versioning. Forks are encouraged to pull upstream when a version ships.

## [0.2.0] — 2026-04-17

### Added
- Branded, Brandhorse-authored README with positioning, target audiences, and an integration story.
- `USAGE.md` — a 5-step walkthrough for populating a fork.
- `CHANGELOG.md` (this file).
- `CONTRIBUTING.md` with guidance for issues and PRs.
- `.github/ISSUE_TEMPLATE/` with asset-library-improvement, folder-suggestion, and case-study-submission templates.
- Folder-level `README.md` stubs in every top-level directory explaining intended contents.
- Enhanced `00_Start-Here/` documentation (asset index with example rows, clearer naming convention, ownership template).
- Social preview image and repo description/topics set via GitHub Settings.

### Changed
- Repo flagged as a GitHub "template repository" — "Use this template" button now available.
- Companion article file-tree synced to actual repo structure.

### Proof points (coming soon)
- Public Brandhorse fork showing a fully populated library.

## [0.1.0] — 2025-10-XX

### Added
- Initial public scaffold: top-level folders, subfolder structure, MIT license, `00_Start-Here/` starter files, placeholder `.gitkeep` files.
